Projet: Vue

Les vues sont le côté front-end du site. En fait sur le projet, ce sont un peu plus que des simples vues : ce sont des templates.
Les vues peuvent contenir de l'HTML classique. Il existe cependant quelques extensions utilisables afin de ne pas intégrer de PHP dedans.

Afficher une variable

Pour afficher une variable qui provient du contrôleur, il suffit, par exemple pour une variable nommée var de faire: {{ var }}

Lien vers un fichier assets

Pour cela, vous pouvez utiliser une fonction assets. Elle construira toute seule le lien vers le fichier.
Par exemple, pour un fichier style.css se trouvant dans assets/css/style.css, on peut utiliser {% assets(css/style.css) %}.
Le Kernel se chargera tout seul de générer le lien pour aller chercher ce fichier.

Lien vers une autre route

Si vous voulez placer une redirection vers une autre page, il est possible d'utiliser la fonction route.
Exemple: si vous voulez rediriger l'utilisateur vers la page http://mon-site.com/forum/message/34/, vous pouvez utiliser dans le code: {% route(/forum/message/34/ %}
Le framework se chargera seul de générer le bon lien.

Inclure une vue dans une autre

Pour palier au manque de l'include() venant de PHP, il existe dans les templates sensiblement la même fonction.
Pour inclure une autre vue, nommée par exemple header.html, il suffit de taper : {% include(header.html) %}
